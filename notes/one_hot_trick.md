# One-Hot Trick (Hashing Trick) in NLP

## Definition

The One-Hot Trick (Hashing Trick) is a text encoding technique that converts words into integer indices using a hash function instead of creating and storing a vocabulary. It is implemented in Keras using the `one_hot()` function and is commonly used before an Embedding layer.

---

## Why Do We Need It?

Machine Learning and Deep Learning models cannot process text directly. Therefore, text must be converted into numerical form.

Sentence:

```text
I love machine learning
```

After applying One-Hot Trick:

```text
[4532, 128, 7821, 3210]
```

Each word is mapped to an integer index generated by a hash function.

---

## Working of One-Hot Trick

### Step 1: Input Sentence

```text
I love machine learning
```

### Step 2: Tokenization

```text
["i", "love", "machine", "learning"]
```

### Step 3: Hash Function

Using a predefined vocabulary size:

```python
vocabulary_size = 10000
```

Words are converted into indices:

```text
i         → 4532
love      → 128
machine   → 7821
learning  → 3210
```

Output:

```text
[4532, 128, 7821, 3210]
```

### Step 4: Padding

Since sentences have different lengths, padding is applied:

```text
[0,0,0,0,4532,128,7821,3210]
```

### Step 5: Embedding Layer

The integer indices are transformed into dense vectors:

```text
4532 → [0.12, -0.43, 0.67, ...]
128  → [0.54, 0.21, -0.11, ...]
```

These vectors are learned during training.

---

## Keras Implementation

```python
from tensorflow.keras.preprocessing.text import one_hot

sentence = "I love machine learning"

one_hot_representation = one_hot(
    sentence,
    10000
)

print(one_hot_representation)
```

Output:

```text
[4532, 128, 7821, 3210]
```

# Normal One-Hot Encoding vs One-Hot Trick

| Feature                       | Normal One-Hot Encoding | One-Hot Trick |
| ----------------------------- | ----------------------- | ------------- |
| Vocabulary Required           | Yes                     | No            |
| Stores Dictionary             | Yes                     | No            |
| Memory Usage                  | High                    | Low           |
| Speed                         | Slower                  | Faster        |
| Collision Possible            | No                      | Yes           |
| Suitable for Large Vocabulary | No                      | Yes           |
| Used Before Embedding         | Rarely                  | Commonly      |

---

## Example of Normal One-Hot Encoding

Vocabulary:

```text
["cat", "dog", "bird"]
```

Representations:

```text
cat  → [1,0,0]
dog  → [0,1,0]
bird → [0,0,1]
```

For a vocabulary of 10,000 words, each vector would have length 10,000, leading to huge memory requirements.

---

## Example of One-Hot Trick

```python
one_hot("cat dog bird", 10000)
```

Output:

```text
[123, 456, 789]
```

Instead of creating three large vectors, only integer IDs are stored.

---

## Hash Collision

A limitation of the One-Hot Trick is hash collision.

Example:

```text
apple  → 345
orange → 345
```

Different words may receive the same index if the vocabulary size is small.

This can introduce ambiguity in the model.

---

## Advantages

1. No need to create a vocabulary.
2. Memory efficient.
3. Fast processing.
4. Suitable for large text datasets.
5. Works effectively with Embedding layers.
6. Easy implementation using Keras.

---

## Disadvantages

1. Hash collisions may occur.
2. Word-to-index mapping is not interpretable.
3. Less accurate than vocabulary-based tokenizers.
4. Semantic relationships between words are not preserved directly.

---

## Complete NLP Pipeline

```text
Text
 ↓
One-Hot Trick
 ↓
Integer Indices
 ↓
Padding
 ↓
Embedding Layer
 ↓
LSTM / GRU / CNN
 ↓
Output Prediction
```

---



**One-Hot Trick (Hashing Trick)** is a text preprocessing technique that uses a hash function to convert words into fixed-range integer indices without explicitly creating a vocabulary. It is memory-efficient, fast, and commonly used before embedding layers in deep learning models for NLP.
